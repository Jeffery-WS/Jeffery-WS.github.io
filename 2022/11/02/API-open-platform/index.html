day1

背景：

  1. 前端开发需要用到后台接口
  2. 使用现有的系统的功能(搏天API)

做一个API接口平台：

  1. 防止攻击（安全性）
  2. 不能随意调用（限制、接通）
  3. 统计调用次数
  4. 计费
  5. 流量保护
  6. API接入

项目介绍

做一个提供API接口调用的平台，用户可以注册登录，开通接口调用权限。用户可以使用接口，并且每次调用会进行统计。管理员可以发布接口、下线接口、接入接口，以及可视化接口的调用情况、数据。

技术选型

  • Umi
  • Umi Request(Axios的封装)

后端

  • Java Spring Boot
  • Spring Boot Starter(SDK开发)
  • ???(网关、限流、日志等实现)

需求分析

  1. 管理员可以对接口信息进行增删改查
  2. 用户可以访问平台，查看接口信息
